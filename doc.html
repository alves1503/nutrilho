Rodando o Nutrilho
no seu iPhone

Guia completo para rodar o projeto React Native com Expo no iOS via VSCode. Siga os passos abaixo na ordem.

iOS only React Native Expo VSCode
1

Pré-requisitos

Certifique-se de ter o ambiente instalado antes de continuar.

Node.js v18 ou superior nodejs.org
VSCode Qualquer versão code.visualstudio.com
Expo Go App no iPhone App Store
Git Para clonar o projeto git-scm.com

O iPhone e o computador precisam estar na mesma rede Wi-Fi para o Expo conseguir se conectar.

2

Baixar o Projeto

Faça o download do arquivo ZIP ou clone via Git.

Download direto Recomendado para iniciantes
Recomendado
Baixar nutrilho.zip

Após baixar, extraia a pasta e abra-a no VSCode com File → Open Folder

Clonar com Git Para quem usa terminal
bash 
git clone https://github.com/ICEI-PUC-Minas-PMV-ADS/pmv-ads-2026-1-e3-proj-mov-t5-nutrilho
cd nutrilho
3

Instalar Dependências

Abra o terminal integrado do VSCode com Ctrl + ` e rode:

bash 
npm install

Isso pode levar alguns minutos na primeira vez. Você verá uma pasta node_modules sendo criada — é normal.

Se preferir usar Yarn:

bash 
yarn install
4

Rodar o Projeto

Inicie o servidor de desenvolvimento do Expo:

bash 
npx expo start
a

O terminal exibirá um QR Code no terminal ou no browser

b

Abra o app Expo Go no iPhone

c

Escaneie o QR Code com a câmera do Expo Go

d

O app abrirá automaticamente no seu iPhone 🎉

Pronto! Qualquer alteração no código será refletida em tempo real no iPhone via Fast Refresh.

5

Problemas Comuns

Soluções para os erros mais frequentes.

Expo Go não consegue se conectar

Verifique que iPhone e computador estão na mesma rede Wi-Fi. Se não funcionar, tente o modo tunnel:

bash 
npx expo start --tunnel
Erro ao rodar npm install

Apague a pasta node_modules e o arquivo package-lock.json, depois reinstale:

bash 
rm -rf node_modules package-lock.json
npm install
App travou ou não atualiza

Limpe o cache do Expo e reinicie:

bash 
npx expo start --clear
iPhone não aparece no Expo Go

Certifique-se de que o Expo Go está atualizado na App Store. Versões muito antigas podem não ser compatíveis com o SDK atual do projeto.
